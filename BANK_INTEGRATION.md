# Banking Integration

This document describes the PostgreSQL banking module and the Python bridge that connects it to the Fitness Club MySQL database.

---

## Overview

```
┌─────────────────────────────┐        ┌──────────────────────────────────┐
│  Fitness Club DB  (MySQL)   │        │   Banking DB  (PostgreSQL)       │
│                             │        │                                  │
│  members                    │  ───►  │  bank_accounts (member)          │
│  trainers                   │  ───►  │  bank_accounts (trainer)         │
│  clubs                      │  ───►  │  bank_accounts (club)            │
│  memberships (active)       │  ──┐   │  transactions                    │
│  trainers (salary)          │  ──┘   │  error_logs                      │
└─────────────────────────────┘        │                                  │
                                       │  loans                           │
       bank_bridge.py                  │  loan_payment_schedule           │
       (Python)                        └──────────────────────────────────┘
```

## Banking Schema (`bank_schema.sql`)

### Tables

| Table | Purpose |
|-------|---------|
| `bank_accounts` | One row per account holder (member / trainer / club). Stores balance and a back-reference to the fitness DB (`fitness_ref_type`, `fitness_ref_id`). |
| `loans` | Credit records linked to a bank account: principal, annual interest rate, term in months. |
| `loan_payment_schedule` | Amortisation schedule rows generated by `generate_payment_schedule()`. |
| `transactions` | Every successfully executed payment. Immutable audit trail. |
| `error_logs` | Failed payment attempts (validation errors, insufficient funds, unexpected exceptions). |

### Stored Procedures

#### `generate_payment_schedule(p_loan_id INT)`

Calculates an **annuity** (equal monthly payment) schedule and inserts rows into `loan_payment_schedule`.

- Deletes any previously generated rows for the same loan.
- Formula: `M = P × r × (1+r)ⁿ / ((1+r)ⁿ – 1)` where `r` is the monthly rate.
- The final instalment is adjusted to absorb rounding drift so the balance reaches exactly zero.

```sql
CALL generate_payment_schedule(1);
SELECT * FROM loan_payment_schedule WHERE loan_id = 1;
```

#### `make_payment(p_debit_account_id, p_credit_account_id, p_amount, p_description, p_reference)`

Executes a transfer between two accounts.

**Validations:**
1. `p_amount` must be positive.
2. Debit and credit accounts must differ.
3. Debit account must exist.
4. Debit account must have sufficient funds.
5. Credit account must exist.

If any validation fails, or an unexpected exception occurs, the payment is **not executed** and a row is written to `error_logs`. No exception is propagated to the caller.

```sql
-- Successful payment
CALL make_payment(4, 1, 75.00, 'Monthly membership fee — Premium', 'fitness:membership:1');

-- Failed payment (insufficient funds) — only an error_log row is created
CALL make_payment(4, 1, 999999.00, 'Test', 'test:ref');

SELECT * FROM transactions ORDER BY executed_at DESC LIMIT 5;
SELECT * FROM error_logs  ORDER BY logged_at  DESC LIMIT 5;
```

## Setting Up the Banking Database

### Prerequisites

- PostgreSQL 13+
- Python 3.8+
- MySQL 5.7+ with the fitness schema already loaded (`schema.sql`)

### 1 — Create the PostgreSQL database

```bash
psql -U postgres -c "CREATE DATABASE bank_db;"
psql -U postgres -d bank_db -f bank_schema.sql
```

### 2 — Install Python dependencies

```bash
pip install mysql-connector-python psycopg2-binary python-dotenv
```

### 3 — Configure environment

```bash
cp .env.example .env
# Edit .env and fill in the credentials for both databases
```

The relevant variables for the bridge:

```
FITNESS_DB_HOST=localhost
FITNESS_DB_PORT=3306
FITNESS_DB_USER=root
FITNESS_DB_PASSWORD=…
FITNESS_DB_NAME=fitness_club_db

BANK_DB_HOST=localhost
BANK_DB_PORT=5432
BANK_DB_USER=postgres
BANK_DB_PASSWORD=…
BANK_DB_NAME=bank_db
```

### 4 — Run the bridge

```bash
python bank_bridge.py
```

## How the Bridge Works (`bank_bridge.py`)

The bridge is a single Python script that:

1. **Reads active memberships** from the fitness MySQL database.
2. For each membership whose monthly fee has **not yet been paid this month**, calls `make_payment` to debit the member's bank account and credit the club's bank account.
3. **Reads all trainers** with a club assignment from the fitness DB.
4. For each trainer, calls `make_payment` to pay the monthly salary from the club's account to the trainer's account.
5. Prints a summary of recent transactions and any errors.

Duplicate-payment protection is provided by checking `transactions` for an existing row with the same `(debit_id, credit_id, reference, month)` before each call to `make_payment`.

### Payment references

| Scenario | Reference format |
|----------|-----------------|
| Membership fee | `fitness:membership:<membership_id>` |
| Trainer salary | `fitness:salary:trainer:<trainer_id>` |

## Automating with cron

To run the bridge automatically on the 1st of every month:

```
0 9 1 * * /usr/bin/python3 /path/to/bank_bridge.py >> /var/log/bank_bridge.log 2>&1
```
